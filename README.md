# Trismegisto

CT-based aorta segmentation and radiomic feature extraction pipeline. The project processes DICOM CT scans, segments the aorta using TotalSegmentator, extracts morphological features (volume, surface area, diameters, eccentricity), and computes radiomic features with PyRadiomics.

---

## Repository Structure

```
Trismegisto/
├── environments/
│   ├── requirements_py39.txt   # Pyradiomics pipeline (Python 3.9)

│   ├── setup_envs.sh           # Linux/macOS setup script
│   └── setup_envs.bat          # Windows setup script
├── scripts/
│   ├── features_extraction.py  # Segmentation + morphological features (Python 3.12)
```

## Two-Environment Architecture

This project uses **two separate Python environments** because `pyradiomics` requires Python ≤ 3.9 while `TotalSegmentator` and the rest of the pipeline support Python 3.12.

| Environment | Python | Key packages | Script |
|-------------|--------|-------------|--------|
| `.venv-py39` | 3.9 | pyradiomics, SimpleITK, numpy==1.26.4 | `scripts/pyradiomics_extraction.py` |
| `.venv-py312` | 3.12 | TotalSegmentator, SimpleITK, scikit-image | `scripts/features_extraction.py` |

## Setup

### Prerequisites

- Python 3.9 installed (e.g. via [pyenv](https://github.com/pyenv/pyenv) or [conda](https://docs.conda.io))
- Python 3.12 installed
- A CUDA-capable GPU is recommended for TotalSegmentator (CPU mode is supported but slow)

### Linux / macOS

```bash
bash environments/setup_envs.sh
```

### Windows

```bat
environments\setup_envs.bat
```

### Manual setup

```bash
# Python 3.9 environment
python3.9 -m venv .venv-py39
source .venv-py39/bin/activate       # Windows: .venv-py39\Scripts\activate
pip install pydicom ipykernel pyradiomics
pip install SimpleITK
pip install "opencv-python==4.9.0.80"
pip install "numpy<2.0.0"
pip install GDCM
pip install pylibjpeg pylibjpeg-libjpeg "numpy==1.26.4"
pip install "pandas==2.2.1"
deactivate

# Python 3.12 environment
python3.12 -m venv .venv-py312
source .venv-py312/bin/activate      # Windows: .venv-py312\Scripts\activate
pip install -r environments/requirements_py312.txt
deactivate
```

## Usage

### 1. Features Extraction (Python 3.12)

Segments the aorta with TotalSegmentator and computes morphological features.

```bash
source .venv-py312/bin/activate

# Edit the paths at the top of the script before running:
#   dicom_path   — folder containing the DICOM series
#   study_name   — patient/study identifier
#   condition    — e.g. "ADX"

python scripts/features_extraction.py
```

Output: `data/features/<condition>/<study_name>_<condition>.csv`

### 2. Pyradiomics Extraction (Python 3.9)

Computes first-order and texture radiomic features from the segmented aorta mask.

```bash
source .venv-py39/bin/activate

# Edit name, condition, dicom_path, and mask_path at the top of the script.

python scripts/pyradiomics_extraction.py
```

Output: `data/features/<condition>/<study_name>_pyradiomics.csv`

## Data Layout

Place your data under `data/` following this structure:

```
data/
├── CT_dicom/
│   └── <condition>/
│       └── <study_name>/       ← DICOM files go here
└── CT_mask/
    └── <condition>/
        └── <study_name>/
            └── aorta.nii.gz    ← generated by TotalSegmentator
```

> **Important:** `data/` is excluded from version control via `.gitignore`. Never commit patient data.

## Pipeline Overview

```
DICOM series
     │
     ▼
TotalSegmentator (Python 3.12)
     │  generates aorta.nii.gz mask
     ▼
Mask post-processing
     │  removes iliac / inlet/outlet artifacts
     ▼
Region segmentation
     │  splits into: ascending arch, descending arch, descending aorta
     ▼
Morphological features (Python 3.12)
     │  volume, surface area, cross-sectional diameters, eccentricity
     ▼
Radiomic features (Python 3.9)
     │  first-order statistics, GLCM, GLRLM, shape features (PyRadiomics)
     ▼
CSV output per study
```

## Notes

- TotalSegmentator uses significant RAM and GPU memory. Use `device="cpu"` only for testing.
- The `fast=False` flag in TotalSegmentator gives higher-quality segmentation at the cost of speed.
- Mask post-processing (inlet/outlet trimming, iliac removal) is tunable via `slices_iniciales_a_borrar` and `slices_superiores_a_modificar` constants in `features_extraction.py`.
